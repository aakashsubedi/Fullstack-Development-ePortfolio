What is HTML?

HyperText Markup Language is the skeleton of every webpage — it gives structure, meaning, and accessibility to content on the internet.

1 / INTRODUCTION

HTML elements & anatomy

HTML elements are defined by tags: an opening tag, content, and a closing tag. Attributes provide additional information — like href for links or alt for images.

HTML 
<article class="post" id="main-post">
  <h2>Understanding the web</h2>
  <p>HTML elements form the <strong>document tree</strong>.</p>
</article>

Start tag

<p> marks beginning

End tag

</p> closes element

Content

Text or nested tags inside

Void elements

<img>, <br> have no closing tag

Attributes like class, id or aria-label make HTML dynamic and accessible. Understanding element hierarchy is the first step toward clean, maintainable pages.

2 / DOCUMENT STRUCTURE

The skeleton of a web page

Every HTML document starts with a !DOCTYPE html and follows a predictable tree of <html>, <head>, and <body>.

HTML 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Document title</title>
</head>
<body>
  <h1>Visible content</h1>
</body>
</html>
<!DOCTYPE html>

Tells the browser to render in standards mode, avoiding quirks.

<html lang="en">

Root element: defines language for accessibility & SEO.

<head>

Metadata, title, links to styles, scripts, and SEO tags.

<body>

Where all visible content lives: text, images, buttons, articles.

📌 Best practice

Always set lang and a unique <title> — this defines the page’s identity for search engines and screen readers.

3 / SEMANTIC HTML

Meaningful markup

Semantic tags describe the role of content: <header>, <nav>, <main>, <article> & more improve readability, SEO and accessibility.

<header>brand + navigation
<main>
<article>Blog post
<section>Group of content
<aside>Sidebar
<footer>Copyright & links
Tag Purpose Accessible role
<header> Introductory content / banner banner
<nav> Primary navigation block navigation
<main> Main content (unique per page) main
<article> Self-contained composition article
<section> Thematic grouping region
<aside> Tangential / sidebar content complementary
<footer> Footer / copyright / info contentinfo
⭐ Why semantics matter

Search engines rank semantic pages higher, screen readers provide landmark navigation, and developers understand layout instantly.

4 / ACCESSIBILITY FIRST

Writing inclusive HTML

Accessibility starts in markup: alt texts, heading hierarchy, and form labels are not optional — they are mandatory for a robust web.

HTML 
<!-- alt text for images -->
<img src="team.jpg" alt="Developers pair programming">
<!-- Descriptive link -->
<a href="/guide">Read the HTML guide</a>
<!-- Proper label + input -->
<label for="email">Email</label>
<input type="email" id="email">

✔️ Practical checklist (human-first)

  • ✓ Every image has meaningful alt text
  • ✓ Headings follow logical order (h1 → h2 → h3)
  • ✓ All form inputs have associated <label>
  • ✓ Color is not the only way to convey meaning
  • ✓ Interactive elements are keyboard accessible
