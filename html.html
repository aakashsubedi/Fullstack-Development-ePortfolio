What is HTML?

HyperText Markup Language is the skeleton of every webpage — it gives structure, meaning, and accessibility to content on the internet.

01 / INTRODUCTION

HTML Elements

HTML elements are the building blocks of web pages. They are defined by a start tag, some content, and an end tag. The browser reads these tags and renders them visually for the user.

HTML 
<h1>Hello World</h1>
<p>This is a paragraph.</p>
<a href="https://example.com">Visit Example</a>
🏷️

Start Tag

Opens an element, e.g. <p>

📝

Content

The text or nested elements inside

🔚

End Tag

Closes an element, e.g. </p>

⚡

Void Elements

Self-closing: <img>, <br>

02 / DOCUMENT STRUCTURE

The HTML Document Structure

Every HTML page follows a standard structure. Understanding each part is essential before writing a single line of code.

HTML 
<!-- Declares the document as HTML5 -->
<!DOCTYPE html>
<html lang="en">

  <!-- Metadata: not visible to users -->
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Page Title</title>
    <link rel="stylesheet" href="styles.css">
  </head>

  <!-- All visible content goes here -->
  <body>
    <h1>Hello, World!</h1>
    <p>This is my first HTML page.</p>
  </body>

</html>

Breaking It Down

<!DOCTYPE html>

Not an HTML element — it's an instruction to the browser. It declares this is an HTML5 document. Without it, browsers may enter quirks mode and render pages inconsistently.

<html lang="en">

The root element that wraps everything. The lang attribute is critical for accessibility — screen readers use it to announce content in the correct language.

<head>

Contains metadata — information about the page, not on the page. Includes the <title> (shown on browser tab), CSS links, fonts, SEO meta tags, and viewport settings.

<body>

Everything inside <body> is rendered on the screen. This is where all your visible content lives — text, images, buttons, navigation, footers, and more.

💡 Best Practice

Always set lang on the <html> element and include a descriptive <title>. These two small steps significantly improve SEO and accessibility for every user.

03 / SEMANTIC HTML

Semantic Tags

Semantic HTML elements clearly describe their meaning to both the browser and the developer. Instead of using generic <div> and <span> everywhere, semantic tags give your HTML structure and meaning.

<header>Logo, Nav
<nav>Navigation links
<main>
<article>Blog post / Card
<section>Content group
<aside>Sidebar
<footer>Links, Copyright

Key Semantic Elements

Tag Purpose Replaces ARIA Role
<header>Site or section header<div id="header">banner
<nav>Navigation links<div id="nav">navigation
<main>Primary page content<div id="content">main
<article>Self-contained content<div class="post">article
<section>Thematic group of content<div class="section">region
<aside>Sidebar / tangential info<div id="sidebar">complementary
<footer>Site or section footer<div id="footer">contentinfo
<figure>Image with caption<div class="image">figure
⚠️ Common Mistake

Using <div> for everything works visually, but screen readers and search engines can't understand your page structure. Always reach for a semantic element first.

✅ Why Semantics Matter

Semantic HTML improves SEO (search engines understand content hierarchy), Accessibility (screen readers navigate by landmarks), and Maintainability (code is self-documenting).

04 / ACCESSIBILITY

Accessibility in HTML

Accessible HTML ensures that all users — including those using screen readers, keyboard navigation, or low-vision tools — can use your webpage effectively. Accessibility starts in HTML, not CSS.

1. Alt Text for Images

Every <img> element should have an alt attribute. Screen readers read the alt text aloud to describe the image.

HTML 
<!-- ✅ Descriptive alt text -->
<img src="team-photo.jpg"
     alt="Our dev team standing in front of the office">

<!-- ✅ Decorative image (empty alt is intentional) -->
<img src="divider.png" alt="" role="presentation">

<!-- ❌ Never skip alt entirely -->
<img src="logo.png"> <!-- Screen readers say "image" -- useless! -->

2. Heading Hierarchy

Use headings in logical order (h1 → h2 → h3). Screen reader users often navigate a page by jumping between headings — a proper hierarchy is like a table of contents.

HTML 
<!-- ✅ Correct heading hierarchy -->
<h1>Page Title</h1>
  <h2>Main Section</h2>
    <h3>Sub-topic</h3>

<!-- ❌ Skipping levels (bad for screen readers) -->
<h1>Page Title</h1>
  <h4>Jumped straight to h4!</h4>

3. Labels & Forms

HTML 
<!-- ✅ Properly associated label -->
<label for="email">Email Address</label>
<input type="email" id="email" name="email"
       required aria-describedby="email-hint">
<span id="email-hint">We'll never share your email.</span>

<!-- ❌ Unlabeled input (screen reader says nothing useful) -->
<input type="email" placeholder="Email">

🧪 Quick Accessibility Checklist

  • ✅ All images have meaningful alt text
  • ✅ Only one <h1> per page
  • ✅ Headings follow logical order (h1→h2→h3)
  • ✅ Every form input has an associated <label>
  • ✅ Links have descriptive text (not "click here")
  • ✅ Interactive elements are keyboard-focusable
  • ✅ Color is not the only way to convey meaning
  • ✅ Language is set on the <html> element

Why These Semantic Choices?

header

Wraps the logo and primary navigation. Screen readers announce it as the banner landmark, so users can jump directly to it.

nav

Both the top nav and footer nav use <nav> with distinct aria-label values so screen readers can differentiate between them.

main

Only one <main> per page. It marks the unique content area — skip-navigation links typically jump directly here for keyboard users.

section

Used for the hero area and posts listing — both are thematically distinct groups. Each has an aria-labelledby pointing to its heading.

article

Each blog post card is an <article> — self-contained, syndication-ready content that makes sense on its own when extracted from the page.

figure

Images are wrapped in <figure> with a <figcaption>. The image has a descriptive aria-label since it's a decorative placeholder here.

aside

The sidebar is an <aside> — tangentially related content (tags, about blurb) that complements but isn't essential to the main content flow.

footer

The page <footer> acts as the contentinfo landmark. Contains copyright, legal links, and a secondary nav — all expected footer content.

06 / FULL CODE

The Complete Blog HTML

Here's the full semantic HTML source for the blog layout above. Notice how reading the tags alone gives you a clear mental model of the page — no CSS needed to understand the structure.

HTML 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>HTML — Web Development Articles</title>
</head>
<body>

  <header role="banner">
    <a href="/">📰 HTML</a>
    <nav aria-label="Main navigation">
      <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/articles">Articles</a></li>
      </ul>
    </nav>
  </header>

  <main role="main">
    <section aria-labelledby="hero-h">
      <h1 id="hero-h">Welcome to HTML</h1>
      <p>Deep dives into modern web development.</p>
    </section>

    <section aria-labelledby="posts-h">
      <h2 id="posts-h">Latest Articles</h2>

      <article aria-labelledby="post1">
        <figure>
          <img src="dom.jpg"
               alt="Abstract code visualization in blue tones">
          <figcaption>Understanding the DOM</figcaption>
        </figure>
        <h3 id="post1">Mastering the DOM</h3>
        <p>Learn how browsers parse HTML...</p>
        <a href="/mastering-dom"
           aria-label="Read: Mastering the DOM">Read more →</a>
      </article>

    </section>
  </main>

  <aside role="complementary" aria-label="Blog sidebar">
    <section>
      <h2>Tags</h2>
      <!-- tag list -->
    </section>
  </aside>

  <footer role="contentinfo">
    <p>© 2026 HTML</p>
    <nav aria-label="Footer navigation">
      <a href="/privacy">Privacy</a>
    </nav>
  </footer>

</body>
</html>
