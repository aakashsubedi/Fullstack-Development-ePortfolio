1 / SELECTORS

CSS Selectors

CSS selectors determine which HTML elements a rule applies to. They range from simple element names to complex structural expressions. Specificity decides which rule wins when multiple rules target the same element.

🎯

Universal & Element

* { }  |  h1 { }

🆔

Class & ID

.card { }  |  #hero { }

🔗

Combinators

ul > li  |  h2 + p  |  div p

✨

Pseudo-classes & Elements

:hover  |  :nth-child()  |  ::before

Specificity — Which Rule Wins?

Every selector gets a score. The higher the score, the more specific it is and it overrides lower-scoring rules targeting the same element.

(0,0,1)

(0,1,1)

CSS — Selectors 
/* 1. Element selector */
h1 { color: navy; }

/* 2. Class selector */
.card { background: #fff; }

/* 3. ID selector (high specificity) */
#hero { font-size: 3rem; }

/* 4. Descendant combinator */
nav a { text-decoration: none; }

/* 5. Child combinator */
ul > li { list-style: disc; }

/* 6. Pseudo-class */
a:hover { color: crimson; }

/* 7. Pseudo-element */
p::first-line { font-weight: 700; }

/* 8. nth-child structural selector */
tr:nth-child(even) { background: #f0f4f8; }
2 / BOX MODEL

The CSS Box Model

Every HTML element is a rectangular box made of four concentric layers. Understanding this model is fundamental to controlling layout and spacing.

MARGIN — space outside the element
BORDER — visible edge of the element
PADDING — space inside the border
CONTENT — your text / image lives here
⚠ box-sizing Gotcha

By default, width: 300px means 300px of content only — padding and border are added on top. Setting box-sizing: border-box includes padding + border inside the declared width, which is almost always what you want. This page uses * { box-sizing: border-box } globally.

CSS — Box Model 
/* Include padding & border inside declared width */
*, *::before, *::after {
  box-sizing: border-box;
}

.card {
  /* Content dimensions */
  width:   300px;
  height:  auto;

  /* Padding — space inside the border */
  padding: 1.5rem;          /* all 4 sides */
  padding: 1rem 2rem;      /* top+bottom | left+right */

  /* Border */
  border:  2px solid #2E5E99;
  border-radius: 10px;

  /* Margin — space outside the border */
  margin: 0 auto;           /* center horizontally */
}

Flexbox Layout

CSS — Flexbox 
.nav-container {
  display:         flex;          /* activate flexbox */
  flex-direction:  row;           /* main axis */
  justify-content: space-between; /* main-axis alignment */
  align-items:     center;        /* cross-axis alignment */
  flex-wrap:       wrap;          /* allow wrapping */
  gap:             1rem;          /* space between items */
}

.flex-item {
  flex: 1 1 auto;  /* grow | shrink | basis */
}

/* Sidebar stacks its children vertically */
.editor-sidebar {
  display:        flex;
  flex-direction: column;
}
💡 Layout Decision — Flexbox Usage

The navbar uses justify-content: space-between to push the logo left and links right. The editor sidebar uses flex-direction: column to stack header, textarea, and preview pane vertically with the textarea consuming remaining space via flex: 1.

4 / CSS GRID

CSS Grid Layout

Grid is a two-dimensional layout model — it controls rows and columns simultaneously. It is the right choice for page-level structures and complex two-axis layouts.

Grid: 3-column template with spanning items

CSS — Grid 
/* Page layout: doc content | sticky sidebar */
.layout-container {
  display:               grid;
  grid-template-columns: 1fr 380px;
  gap:                   2.5rem;
  align-items:           start;
}

/* Responsive card grid — no media queries needed */
.card-grid {
  display:               grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: 1rem;
}

/* Spanning columns and rows */
.featured {
  grid-column: span 2;
  grid-row:    span 2;
}

Content-driven sizing✓ Yes~ Sometimes

Implicit rows/columns✗ No✓ Yes

📐 Layout Decisions on This Page

Page level: CSS Grid (1fr 380px) — two columns, explicit and predictable.
Navbar: Flexbox space-between — one axis, distributes space.
Card grid: repeat(auto-fit, minmax(200px, 1fr)) — self-responsive, zero media queries needed.
Editor sidebar: Flexbox column — stacks panels with flex: 1 filling remaining height.

5 / RESPONSIVE DESIGN

Responsive Design & Media Queries

Responsive design ensures your page looks and works great on every screen. The core tool is the media query — CSS rules that only apply when conditions (like viewport width) are met.

Active Breakpoint (resize your window to see it change)

📱 < 480px
📱 < 600px
📟 < 900px
💻 < 1024px
🖥 ≥ 1024px
CSS — Media Queries (used on this page) 
/* ── Desktop: two-column grid (default) ── */
.layout-container {
  grid-template-columns: 1fr 380px;
}

/* ── Tablet ≤ 1024px: collapse to single column ── */
@media (max-width: 1024px) {
  .layout-container {
    grid-template-columns: 1fr;
  }
  .editor-sidebar {
    position: static;  /* unstick on small screens */
    height: 700px;
  }
  .before-after {
    grid-template-columns: 1fr;
  }
}

/* ── Mobile nav ≤ 900px: hamburger menu ── */
@media (max-width: 900px) {
  .menu-toggle { display: flex; }
  .nav-menu    { position: fixed; left: -100%; }
  .nav-menu.active { left: 0; }
}

/* ── Small mobile ≤ 600px ── */
@media (max-width: 600px) {
  .card-grid { grid-template-columns: 1fr 1fr; }
  .hero      { padding: 3rem 1rem; }
}

/* ── Fluid typography — no breakpoints needed ── */
h1 { font-size: clamp(1.8rem, 5vw, 3.6rem); }
