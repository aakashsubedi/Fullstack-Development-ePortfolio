Section 01

What is Full Stack Development?

Understanding the complete picture of how websites and web applications are built.


Full Stack Development refers to the practice of developing both the frontend (what the user sees in their browser) and the backend (the server-side logic and processing), along with database management — all together as one complete web solution. A Full Stack Developer has knowledge and skills across all three layers of a web application, enabling them to build and maintain an entire project from start to finish.


Think of a restaurant: The frontend is the dining area — what customers see and interact with. The backend is the kitchen — where the food is prepared (data is processed). The database is the pantry — where all the ingredients (data) are stored. A full stack developer manages all three.

Section 02

The Three Layers of a Web Application

Every web application is built on three main layers that each play a distinct role.


Layer 1

Frontend

The frontend is everything the user directly sees and interacts with in their web browser. It is built using three core technologies:

HTML CSS JavaScript
  • HTML – Creates the structure and content (headings, paragraphs, buttons, forms).
  • CSS – Styles the page: colors, fonts, layout, and responsiveness.
  • JavaScript – Adds interactivity: click events, animations, form validation, API calls.
Layer 2

Backend

The backend is the server-side of the application — invisible to the user but essential. It handles business logic, authentication, and communication with the database.

PHP Apache / Nginx REST API
  • Receives requests sent from the browser (frontend).
  • Processes the data and applies business rules.
  • Talks to the database to read or write data.
  • Sends a response back to the browser.
Layer 3

Database

The database is where all data is permanently stored and organised. SQL (Structured Query Language) is used to communicate with relational databases like MySQL.

MySQL SQL Tables / Rows
  • SELECT – Retrieve data from a table.
  • INSERT – Add new data records.
  • UPDATE – Modify existing records.
  • DELETE – Remove records from the database.

Section 04

Example: Student Registration Form

Let's see how all three layers work together in a real, simple project — a student registration form.


Frontend — The HTML Form

The user sees and fills in a form with their name, email, and student ID.

<!-- HTML Form --> <form action="register.php" method="POST"> <input name="name" placeholder="Your Name"> <input name="email" type="email"> <button>Register</button> </form>

Backend — The PHP Script

PHP receives the form data, validates it, and saves it to the database.

// register.php (Backend) $name = $_POST['name']; $email = $_POST['email']; // Connect to DB $conn = mysqli_connect( "localhost","root","","uni" ); // Insert into DB $sql = "INSERT INTO students (name, email) VALUES ('$name','$email')"; mysqli_query($conn, $sql);

Database — MySQL Table

Data is stored permanently in an SQL table inside the MySQL database.

-- Create the table CREATE TABLE students ( id INT AUTO_INCREMENT PRIMARY KEY, name VARCHAR(100), email VARCHAR(100) ); -- View stored records SELECT * FROM students;
Section 05

How Data Flows:

When a user submits a form, here is exactly what happens — from click to response.


1

User Fills & Submits the Form (Frontend)

The user types their name and email in the HTML form and clicks Submit. JavaScript can validate the fields first (e.g., check the email format is correct). Then the browser packages the data and sends an HTTP POST request to the server.

POST /register.php → name=Ali&email=ali@uni.edu
2

Server Receives the Request (Backend — PHP)

The web server (Apache/Nginx) receives the request and hands it to the PHP script. PHP reads the submitted data, applies business logic (validation, security checks), and prepares an SQL query to store or retrieve the data.

$name = $_POST['name']; // "Ali" $email = $_POST['email']; // "ali@uni.edu"
3

Data is Stored in the Database (MySQL / SQL)

PHP sends an SQL INSERT query to the MySQL database. The database validates the query, writes the record to the correct table, and confirms success back to PHP.

INSERT INTO students (name, email) VALUES ('Ali', 'ali@uni.edu'); -- Database confirms: 1 row inserted
4

Response Sent Back to the User (Backend to Frontend)

PHP generates an HTML response (e.g., a success message page or JSON data) and sends it back through the server. The browser receives it and displays the result to the user — the full cycle is complete.

echo "Registration successful! Welcome, $name.";

Bonus: The Complete Request–Response Cycle

Here is the entire journey of a form submission visualised as a simple flow:

User submits form
→
HTTP POST Request
→
PHP processes data
→

→
Response sent back

Why does this matter?
  1. The frontend never directly touches the database — it always goes through the backend. This keeps data secure.
  2. The backend acts as a gatekeeper, validating and sanitising data before it is stored.
  3. The database only accepts properly formatted SQL queries — it does not care about HTML or JS.
  4. This separation of concerns makes full stack applications easier to maintain, scale, and secure.

Conclusion

Full Stack Development is the practice of working across all three layers of a web application — the Frontend, Backend, and Database — to deliver a complete, functional product.

Using HTML, CSS, and JavaScript on the frontend, PHP on the backend, and MySQL (SQL) for the database, a full stack developer can build everything from a simple registration form to a complex web platform.

Understanding how these layers communicate — through HTTP requests, PHP scripts, and SQL queries — is the foundation of modern web development.
